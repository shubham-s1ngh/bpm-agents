Rules are matched conjunctively. If you configure `errorTypes=JOB_NO_RETRIES`, `httpStatusCodes=500`, and `messageContains=payment-charge`, the rule applies only when all of those conditions appear in the incident evidence. The selected resolution mode then decides whether retry is allowed, blocked, or skipped.
